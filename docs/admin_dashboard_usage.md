# 관리자 대시보드 사용 가이드

## 🚀 접속 방법

서버가 실행 중인 상태에서 브라우저에서 다음 URL로 접속하세요:

```
http://localhost:8080/admin/
```

또는

```
http://localhost:8080/admin/index.html
```

---

## 📋 기능

### 1. 로그인

- 관리자 이메일과 비밀번호로 로그인
- 로그인 정보는 브라우저의 localStorage에 저장됩니다
- 이미 로그인되어 있으면 자동으로 대시보드로 이동

### 2. 대시보드 (인증 신청 목록)

**주요 기능:**
- 인증 신청 목록 조회
- 상태별 필터링 (대기 중, 승인됨, 전체)
- 통계 카드 (대기 중, 승인됨, 전체 건수)
- 페이지네이션
- 새로고침 버튼

**사용 방법:**
1. 상태 필터를 선택하여 원하는 상태의 인증 신청만 조회
2. 인증 신청 카드를 클릭하면 상세 페이지로 이동
3. "새로고침" 버튼으로 최신 목록 업데이트

### 3. 상세 페이지

**주요 기능:**
- 인증 신청 상세 정보 확인
- 자격증 이미지 확인 (클릭하면 확대)
- 신분증 이미지 확인 (있는 경우)
- 승인/거부 기능

**사용 방법:**
1. 대기 중 상태의 인증 신청만 승인/거부 가능
2. 승인: "승인" 버튼 클릭 → 메모 입력 (선택) → 승인
3. 거부: "거부" 버튼 클릭 → 거부 사유 입력 (필수) → 메모 입력 (선택) → 거부
4. 이미지를 클릭하면 모달로 확대된 이미지 확인 가능

## 🔐 보안

- 모든 API 호출은 인증 토큰이 필요합니다
- 토큰이 만료되거나 권한이 없으면 자동으로 로그인 페이지로 리다이렉트됩니다
- 로그아웃 시 localStorage의 토큰이 삭제됩니다

---

## 🎨 UI 특징

- 반응형 디자인 (모바일/태블릿/PC 지원)
- 깔끔하고 직관적인 인터페이스
- 로딩 상태 표시
- 에러 메시지 표시
- 상태별 색상 구분 (대기 중: 노란색, 승인됨: 초록색, 거부됨: 빨간색)

## 🛠 문제 해결

### 로그인 후 대시보드가 안 보여요
- 브라우저 콘솔(F12)에서 에러 확인
- 서버가 실행 중인지 확인
- 관리자 이메일이 `.env` 파일의 `ADMIN_EMAILS`에 설정되어 있는지 확인

### 이미지가 안 보여요
- 이미지 URL이 올바른지 확인
- 서버의 CORS 설정 확인
- 브라우저 콘솔에서 네트워크 에러 확인

### API 호출이 실패해요
- 서버가 실행 중인지 확인
- 토큰이 만료되었는지 확인 (로그아웃 후 다시 로그인)
- 브라우저 콘솔에서 에러 메시지 확인

---

## 📝 참고

- 관리자 계정은 서버의 `.env` 파일에 `ADMIN_EMAILS`로 설정된 이메일 주소입니다
- 여러 관리자가 필요한 경우 쉼표로 구분하여 추가할 수 있습니다
- 예: `ADMIN_EMAILS=admin1@example.com,admin2@example.com`
